# Specification

Nous avons fait le choix de nous rapprocher du comportement natif d'un input, en ajoutant en plus la validation à la volée.
Nous avons donc utiliser la méthode checkValidity lors d'un changement de valeur ainsi que lorsque le champ est touché.
L'interet étant de rapporter la non validité grâce au onInvalid qui est appelé si le checkValidity rapporte un champ invalide.

Pour faire de la gestion de message d'erreur customisée, nous pouvons utiliser la propriété `validation` que l'on transmet au champ.
Le retour de cette méthode est utilisée pour mettre à jour la valeur du message d'erreur de l'input.
